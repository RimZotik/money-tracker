# Money Tracker

Десктопное приложение для учёта личных финансов. Tauri 2 + Vite + React + TypeScript, база — SQLite.

## Главный принцип

**Остатки нигде не хранятся.** В базе лежит только журнал операций; баланс счёта,
долг по кредиту и любая аналитика вычисляются из него запросом. Поэтому правка
операции пятилетней давности автоматически пересчитывает всё до сегодняшнего дня —
пересчитывать просто нечего.

## Модель данных

Три типа операций, и разница между ними принципиальна:

- **Приход** — деньги приходят на счёт извне.
- **Расход** — деньги уходят со счёта наружу.
- **Перевод** — движение между своими счетами. **Не доход и не расход.** Именно
  смешение переводов с доходом ломало учёт в старой Excel-таблице.

**Погашение кредита** — это расход с указанным кредитом, где сумма разложена на
`principal_part` (тело долга, уменьшает долг, но тратой не является) и
`interest_part` (проценты — вот это настоящий расход).

**Кредитная карта** — счёт типа `credit`. Её долг — просто отрицательный остаток,
покупка по ней — обычный расход, а погашение — перевод с дебетовой карты.

**Возврат покупки** (`is_refund`) привязан к исходному расходу через `refund_of`.
В режиме «зачитывать возвраты» он гасит этот расход, а не считается доходом:
купил подписку за 449 и вернул — потрачено 0, а не «расход 449 и доход 449».

**Корректировка остатка** — служебная операция в категории с `is_service = 1`.
Нужна, когда детализация выписки не сходится с остатком банка (часть комиссий он
списывает, не показывая строкой). В журнале и аналитике такие операции скрыты.

Счета: `bank`, `credit` (лимит, ставка, грейс), `crypto` (сеть + адрес), `cash`,
`deposit` (вклад, брокерский счёт, счёт на ставках — живёт на странице «Вклады»).

## Запуск

Нужны Node.js, Rust и **MSVC Build Tools** (без них Rust не слинкует бинарник):

```
winget install --id Microsoft.VisualStudio.2022.BuildTools --override "--wait --quiet --add Microsoft.VisualStudio.Workload.VCTools --includeRecommended"
```

Дальше:

```
npm install
npm run tauri dev      # разработка
npm run tauri build    # сборка .msi под Windows
```

База создаётся при первом запуске в `%APPDATA%\com.rimzo.money-tracker\money.db`,
вместе со стартовыми категориями и правилами автокатегоризации.

## Импорт выписок

`tools/import_statements.py` разбирает выписки и заливает их в базу приложения.
Поддержаны Сбербанк (PDF), Озон Банк (PDF), Т-Банк (CSV), ВТБ (CSV).

```
python tools/import_statements.py            # разбор и отчёт, база не трогается
python tools/import_statements.py --write    # записать
python tools/import_statements.py --write --reset   # снести базу и залить заново
```

Нужен `pypdf` (`pip install pypdf`). Запускать из корня проекта, где лежат выписки.

Что импортёр делает неочевидного:

1. **Сначала собирает операции всех банков, и только потом ищет переводы.**
   Поэтому перевод Сбер → Озон виден с обеих сторон и склеивается в один
   Transfer, а не в «расход плюс доход».
2. **Пропускает изменения кредитного лимита.** Строки вида «Прочие операции
   +57 000» без описания на кредитке — это не деньги, а рост лимита (17k → 29k →
   58k → 115k). Импорт их доходом завысил бы учёт на 115 000 ₽.
3. **Сверяется по последнему остатку в детализации, а не по шапке выписки** —
   шапка бывает устаревшей (у карты 1510 она показывает 8 105,53 при фактических
   6 508,58). Расхождение закрывается служебной корректировкой.
4. **Находит возвраты** двумя способами: по коду авторизации (банк отдаёт возврат
   тем же кодом) и по метке «Возврат покупки» с совпадением суммы и мерчанта.
5. **Категорию берёт по мерчанту**, а не по категории банка: у Сбера половина
   оборота лежит в «Прочих операциях» и «Переводах».

Повторный запуск безопасен: у каждой операции есть `ext_id`, дубликаты отсекаются.

## Структура

```
src/
  api.ts                команды бэкенда + форматирование денег
  cache.ts              кэш ответов: без него переключение страниц подвисало
  store.tsx             справочники, индикатор загрузки, прогрев данных
  components/
    TxModal             универсальная форма: приход / расход / перевод
    DateRangePicker     календарь с выбором диапазона и масштабом месяц/год
    AccountPicker       мультивыбор счетов
    usePopover          держит выпадающие панели внутри окна
  pages/                Дашборд, Операции, Счета, Кредиты, Вклады, Проекты, Настройки
src-tauri/src/
  schema.sql            схема БД, включая вычисляемые VIEW остатков
  seed.sql              категории и правила автокатегоризации
  commands.rs           команды, доступные фронтенду
tools/
  import_statements.py  импорт: сбор, поиск переводов и возвратов, сверка
  banks/                парсер на каждый банк
```
